# Proyectos desarrollados

## 1. Sistema de Hotel https://hotel.cuneusdata.com/

### Descripción general

Se desarrolló un sistema de gestión hotelera pensado para centralizar la operación diaria del establecimiento. El objetivo principal fue ordenar procesos que normalmente se manejan de forma manual o dispersa, como el control de habitaciones, reservas, estado de ocupación y registro de huéspedes.

### Funcionalidades principales

* Gestión de habitaciones con estados como disponibles, ocupadas, reservadas o en limpieza.
* Registro de huéspedes y empresas asociadas.
* Control de disponibilidad en tiempo real.
* Organización de reservas y asignación de habitaciones.
* Visualización clara del estado operativo del hotel.
* Base preparada para ampliaciones futuras, como facturación, historial de estadías y reportes.

### Objetivo del sistema

Mejorar la organización interna del hotel, reducir errores operativos y facilitar el acceso a la información clave desde una única plataforma.

### Valor aportado

Este sistema permite que la administración tenga una visión mucho más clara del negocio y que el personal trabaje con procesos más ordenados, rápidos y confiables.

## 2. Sistema de Restaurant (en desarrollo)

### Descripción general

Se desarrolló un sistema gastronómico orientado a la operación diaria de restaurantes, con foco en la toma de pedidos, gestión de mesas, comandas e impresión para cocina. La idea fue construir una base sólida para un sistema moderno, rápido y adaptable al flujo real del salón.

### Funcionalidades principales

* Gestión de mesas y estado de ocupación.
* Creación de pedidos por mesa.
* Agregado de productos, cantidades y notas especiales.
* Resumen del pedido con totales claros.
* Lógica de comandas para cocina.
* Preparación para impresión automática de tickets en comandera.
* Base para incorporar módulos como cocina (KDS), caja, stock e inventario.
* Interfaz pensada para uso ágil en contextos de alta rotación.

### Objetivo del sistema

Digitalizar y ordenar el flujo operativo del restaurante, minimizando errores en pedidos y mejorando la comunicación entre salón y cocina.

### Valor aportado

El sistema permite trabajar con más velocidad, más control y menos caos. En gastronomía eso vale oro, porque donde hay papelitos volando, siempre termina explotando algo.

## 3. Sistema Comercial para Vendedores https://www.cuneusdata.com/cuneus-comercial

### Descripción general

Se diseñó un sistema comercial orientado a la gestión de vendedores, clientes, ventas y seguimiento del rendimiento comercial. Está pensado para empresas con equipos de venta que necesitan control, visibilidad y análisis para tomar mejores decisiones.

### Funcionalidades principales

* Gestión de vendedores y clientes.
* Consulta de ventas por período.
* Seguimiento de rendimiento comercial.
* Visualización de comisiones y resultados.
* Segmentación geográfica y comercial.
* Análisis de clientes activos, inactivos y oportunidades de reactivación.
* Integración con dashboards para seguimiento estratégico.
* Base para automatización de consultas mediante chatbot.

### Objetivo del sistema

Dar herramientas tanto a la empresa como a los vendedores para entender mejor su desempeño, detectar oportunidades y optimizar la gestión comercial.

### Valor aportado

Este sistema transforma datos sueltos en información útil. En vez de vender a ciegas, se empieza a vender con criterio.

## 4. Bot para Agenda Médica +54 3417199019

### Descripción general

Se desarrolló un bot conversacional orientado a la gestión de turnos médicos. El sistema permite que pacientes puedan consultar disponibilidad, elegir profesionales, seleccionar días y horarios, y avanzar en el proceso de reserva de forma automatizada.

### Funcionalidades principales

* Atención automatizada por mensajería.
* Selección de especialidad o profesional.
* Consulta de días disponibles.
* Visualización de horarios disponibles.
* Reserva de turnos de forma guiada.
* Navegación conversacional con menús y opciones de retroceso.
* Integración con lógica de agenda y disponibilidad.
* Preparación para trabajar en entornos internos o productivos.

### Objetivo del sistema

Automatizar el proceso de asignación de turnos para reducir carga administrativa, mejorar la experiencia del paciente y agilizar la atención.

### Valor aportado

El bot permite atender consultas repetitivas de forma automática, ordenada y escalable. Básicamente, hace el trabajo pesado sin cansarse ni pedir café.

## 7. Enfoque general de los proyectos

Todos estos desarrollos comparten una misma lógica de trabajo:

* Resolver problemas reales.
* Ordenar procesos.
* Centralizar información.
* Ahorrar tiempo operativo.
* Escalar sobre bases sólidas.
* Combinar sistema, automatización y análisis de datos.

No fueron proyectos hechos “porque sí”. Fueron construidos para atacar necesidades concretas y generar valor real en la operación diaria de distintos negocios.

---

## Cierre

En conjunto, estos proyectos representan una línea de trabajo orientada al desarrollo de soluciones digitales para gestión, automatización y análisis de datos en distintos rubros: hotelería, gastronomía, salud y ventas.

Cada sistema fue pensado con una lógica práctica: que sirva, que ordene y que permita crecer. Porque un sistema que se ve lindo pero no resuelve nada es apenas un adorno caro.
